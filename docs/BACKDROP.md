# Backdrop Asset Documentation

## What it is

The full-screen backdrop is an **original vector illustration** of a
high-desert valley at dusk: layered mesas, red-rock canyon walls, a small
creek catching the last light, washes, and sage across the valley floor. It is
inspired by the general character of the Short Creek region of Northern
Arizona / Southern Utah.

It is **artwork, not a photograph**. It does not depict any specific place,
landmark, property, road, or boundary, and contains no text, buildings, or
anything that could be read as real-estate information.

## Technical details

| Property | Value |
| --- | --- |
| Original asset | Inline `<svg class="scene">` in `index.html` (single source) |
| Optimized web asset | Same — SVG is the delivery format (~6 KB within the HTML) |
| Design canvas | 1600 × 900 viewBox, `preserveAspectRatio="xMidYMid slice"` (covers any viewport, any DPI, no responsive variants needed) |
| File formats | SVG (backdrop), PNG (social image `assets/og-image.png`, 1200 × 630, ~440 KB — fetched only by link previews, never by page visitors) |
| Compression | None needed; the SVG is smaller than any raster/video equivalent |
| Poster / fallback | Not applicable — the SVG itself is the lightweight static asset |

## Animation, mobile, and reduced motion

- All motion lives in `assets/styles.css` under
  `@media (prefers-reduced-motion: no-preference)`: slow scene zoom, layer
  parallax, cloud drift, glow breathing, one-time text fade-in.
- **Reduced motion:** every animation is disabled; the page is fully static.
- **Mobile:** the same SVG serves all devices (vector, so no extra data).
  The `slice` fitting crops the wide composition to portrait gracefully; key
  elements (sun glow, creek) sit near center. There is no video, so there is
  no autoplay or data-usage concern on cellular connections.

## How to replace the backdrop later

1. Produce the new asset (photo, generated image, or video) with confirmed
   commercial rights, and document it in this file.
2. **Image:** in `index.html`, replace the `<svg class="scene">…</svg>` inside
   `<div class="backdrop">` with an `<img class="scene" src="…" alt=""
   fetchpriority="high">`. Provide AVIF/WebP sources via `<picture>` and 1×/2×
   sizes. The `.scene` CSS (cover sizing + slow zoom) still applies; keep the
   `.scrim` overlay for text contrast.
3. **Video:** replace the SVG with
   `<video class="scene" autoplay muted loop playsinline poster="…" preload="metadata">`
   plus a compressed MP4/WebM (target ≤ 4 MB), keep the poster as fallback,
   and hide the video (show the poster) under `prefers-reduced-motion` and on
   small screens via a media query.
4. Regenerate `assets/og-image.png` (screenshot the page at 1200 × 630).
5. Commit and push to `main` — the mirror updates both domains automatically.
